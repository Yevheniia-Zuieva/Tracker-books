# Статичний аналіз та лінтинг

У цьому проєкті використовуються інструменти статичного аналізу коду для підтримання високих стандартів читабельності, запобігання логічним помилкам та забезпечення єдиного стилю програмування. 

Оскільки проєкт розділено на Backend (Django) та Frontend (React), використовуються два відповідних лінтери.

## 1. Обрані лінтери та причини вибору

### Backend (Python)
**Обраний лінтер:** `Ruff`
* **Причини вибору:** Ruff — це сучасний, надзвичайно швидкий лінтер, написаний на Rust. Він замінює собою одразу кілька класичних інструментів (Flake8, Pylint, Isort). Він ефективно знаходить невикористані змінні, порушення PEP 8 та автоматично сортує імпорти.

### Frontend (JavaScript/React)
**Обраний лінтер:** `ESLint`
* **Причини вибору:** Це індустріальний стандарт для JavaScript та React-додатків. Завдяки плагінам (таким як `eslint-plugin-react-hooks`), він дозволяє перевіряти не лише синтаксис, але й специфічні правила хуків React, запобігаючи витокам пам'яті та помилкам стану.

## 2. Базові правила та їх пояснення

### Правила Ruff (Backend)
Конфігурація знаходиться у файлі `pyproject.toml`.
* **`select = ["E", "F", "I"]`**:
  * **E (Errors):** Перевірка стилю коду згідно з PEP 8 (наприклад, правильні відступи, пробіли навколо операторів).
  * **F (Pyflakes):** Пошук логічних помилок (невикористані імпорти, використання неоголошених змінних).
  * **I (Isort):** Автоматичне сортування імпортів в алфавітному порядку та групування за типом (вбудовані, сторонні, локальні).
* **`line-length = 150`**: Збільшено стандартну довжину рядка (80) до 150 символів, щоб дозволити більш читабельні ланцюжки ORM-запитів Django.
* **Ігнорування (`exclude`)**: Директорії `migrations/` та `venv/` ігноруються, оскільки вони містять автозгенерований код або сторонні бібліотеки.

### Правила ESLint (Frontend)
Конфігурація базується на стандартах Vite + React.
* Заборона використання невикористаних змінних (`no-unused-vars`).
* Обов'язкова перевірка залежностей у масивах React-хуків (наприклад, `useEffect`).

## 3. Інструкція з запуску лінтера

### Для Backend (Python)
1. Перейдіть до папки бекенду: `cd backend`
2. Переконайтеся, що віртуальне середовище активовано.
3. Щоб перевірити код на помилки, виконайте:
   ```bash
   ruff check .
4. Щоб автоматично виправити безпечні помилки, виконайте:
    ```bash
    ruff check . --fix

### Для Frontend (React)
1. Перейдіть до папки фронтенду: `cd frontend`
2. Переконайтеся, що встановлені всі залежності:
    ```bash
    npm install
3. Щоб запустити перевірку всього коду на помилки, виконайте:
    ```bash
    npm run lint
4. Для автоматичного виправлення синтаксичних та стилістичних помилок:
    ```bash
    npm run lint -- --fix
