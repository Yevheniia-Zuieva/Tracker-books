# Інструкція з автоматичної генерації документації проєкту

Цей документ описує процес налаштування та генерації технічної документації для серверної (Backend) та клієнтської (Frontend) частин проєкту. 

Документація генерується безпосередньо з коментарів у вихідному коді.

---

## 1. Backend (Python / Django)

Для генерації документації серверної частини використовується інструмент **`pdoc`**. Він автоматично аналізує docstrings у форматі *Google Style* та створює зручний HTML-інтерфейс.

Оскільки проєкт побудований на фреймворку Django, звичайний запуск `pdoc` через командний рядок може викликати помилку `AppRegistryNotReady` (оскільки моделі завантажуються до ініціалізації середовища). Тому для генерації використовується спеціальний скрипт.

### 1.1. Встановлення залежностей
Переконайтеся, що ви знаходитесь у папці `backend`, а ваше віртуальне середовище (`venv`) активоване.

```bash
cd backend
pip install pdoc
```

### 1.2. Скрипт генерації (`pdoc_init.py`)
У корені папки `backend` створено файл `pdoc_init.py`, який завантажує налаштування Django перед запуском парсера:

```python
import os
import django
import pdoc
from pathlib import Path

# Вказуємо шлях до налаштувань проєкту
os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'backend.settings')

# Ініціалізуємо Django для завантаження моделей
django.setup()

# Конвертуємо шлях у об'єкт Path
output_path = Path("../docs/backend_html")

# Запускаємо генерацію документації для додатку tracker
pdoc.pdoc("./tracker", output_directory=output_path)
print("Документацію бекенду успішно згенеровано у docs/backend_html")
```

### 1.3. Запуск генерації
Щоб згенерувати документацію, виконайте команду:

```bash
python pdoc_init.py
```

**Результат:** HTML-файли документації будуть збережені у директорії `docs/backend_html/`. Відкрийте файл `index.html` у цій папці через будь-який браузер для перегляду.

---

## 2. Frontend (React / JavaScript)

Для генерації документації клієнтської частини використовується стандартний інструмент **`JSDoc`**. Він аналізує спеціальні коментарі (`/** ... */`) у React-компонентах, утилітах та API-сервісах.

### 2.1. Встановлення залежностей
Перейдіть до папки `frontend` та встановіть `jsdoc` як залежність для розробки (dev dependency):

```bash
cd ../frontend
npm install --save-dev jsdoc
```

### 2.2. Запуск генерації
Для автоматичного сканування всіх файлів у папці `src` та створення документації виконайте команду:

```bash
npx jsdoc -r src -d ../docs/frontend_html
```

**Пояснення параметрів:**
* `-r src`: рекурсивний пошук коментарів JSDoc у всіх вкладених файлах та папках вихідного коду (`src`).
* `-d ../docs/frontend_html`: збереження згенерованих файлів у спільну папку документації на рівень вище.

**Результат:** HTML-файли документації будуть збережені у директорії `docs/frontend_html/`. Відкрийте файл `index.html` для перегляду структури фронтенду.

---

## 3. Важливі примітки (Git та збереження)

Автоматично згенеровані HTML-файли документації (їх можуть бути сотні) **не повинні** зберігатися у системі контролю версій Git. 

Для цього у кореневому файлі `.gitignore` налаштовано наступні правила ігнорування:

```text
# Ігнорування згенерованої HTML-документації
docs/backend_html/
docs/frontend_html/

# Ігнорування допоміжного скрипта ініціалізації бекенду
pdoc_init.py
```
